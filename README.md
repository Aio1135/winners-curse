# 승자의 저주 (Winner's Curse)

AI 입찰자를 상대로 경매 이론을 공략하는 싱글플레이 웹 게임.

스테이지마다 다른 경매 방식으로 AI들과 경쟁하고, 매 라운드가 끝나면 **복기 화면**에서
전원의 감정치·입찰 근거가 공개됩니다. "아, 그래서 걔가 그랬구나"를 깨닫는 순간이 이 게임의 심장입니다.

> 낙찰했는데 알고 보니 진짜 가치보다 비싸게 샀다 — 그것이 **승자의 저주**.
> 가장 후하게 평가한 사람이 이기기 마련이라, 이겼다는 사실 자체가 과대평가의 신호입니다.

## 실행

```bash
npm install
npm run dev     # http://localhost:5173
```

```bash
npm test        # 엔진 로직 테스트 (Vitest)
npm run build   # 타입 검사 + 정적 빌드 (dist/)
```

## 게임 방법

1. **브리핑** — 경매 방식, 예산, 클리어 조건, 오늘의 상대(AI 성격 힌트)를 확인합니다.
2. **아이템 공개** — 내 감정치가 제시됩니다. 단, 오차(±10~40%)가 있습니다. 진짜 가치는 아무도 모릅니다.
3. **판단** — 감정 의뢰(코인 1개로 오차를 ±10%로 축소), 패스, 입찰 참여 중 선택합니다.
4. **입찰** — 방식별로 긴장의 포인트가 다릅니다 (아래 표).
5. **정산·복기** — 진짜 가치가 공개되고, 전원의 감정치·입찰 근거·판별 피드백이 표시됩니다.
6. **결과** — 확보한 진짜 가치 합계가 목표를 넘으면 클리어. 별점(★~★★★)과 함께 다음 스테이지가 열립니다.

### 경매 방식 4종

| 방식 | 룰 | 이론 포인트 |
|---|---|---|
| 영국식 | 호가가 자동 상승, 마지막 1인이 그 호가에 낙찰 | 차순위 한도 근처에서 낙찰된다 |
| 네덜란드식 | 가격이 하강, 먼저 누르는 사람이 가져감 | 기다림 = 이득이지만 뺏길 위험 |
| 비공개 1가 | 동시 제출, 최고가가 자기 입찰액 지불 | 감정치보다 낮춰 부르는 게 합리적 (bid shading) |
| Vickrey (2가) | 동시 제출, 최고가가 2등 가격 지불 | 감정치 그대로가 (약)우월전략 |

### AI 입찰자 5종

| | 이름 | 성향 |
|---|---|---|
| 🧐 | 감정사 | 감정치 그대로. 기준선 |
| 🦏 | 불도저 | 감정치에 15~30% 얹는다. 연패하면 더 세진다. 승자의 저주 단골 |
| 🪙 | 구두쇠 | 감정치의 8할·예산의 3할이 상한. 막판까지 빈손이면 지갑을 연다 |
| 🎯 | 스나이퍼 | 실시간 경매에서 한도는 감정치의 95%, 행동은 끝까지 숨긴다 |
| 🕶️ | 담합조 | 2인 1조. 감정치 높은 쪽만 나서서 경쟁자 수를 속인다 |

일부 AI는 **선호 카테고리**(브리핑에 "OO 애호가 ❤️"로 표시)가 있어 그 물건엔 더 공격적으로 나옵니다.
모든 행동은 복기 대사에서 정직하게 공개됩니다.

### 스테이지 커리큘럼 (8개)

| # | 제목 | 방식 | 배우는 것 |
|---|---|---|---|
| 1 | 첫 망치소리 | 영국식 | 기본 룰, 차순위 낙찰 감각 |
| 2 | 봉투 속 숫자 | 비공개 1가 | bid shading |
| 3 | 정직의 역설 | Vickrey | 진실 입찰이 최적 |
| 4 | 내려가는 시계 | 네덜란드식 | 타이밍 vs 탐욕 |
| 5 | 승자의 저주 | 비공개 1가 | 공통가치 함정 (감정 오차 ±40%) |
| 6 | 짬짜미 | 영국식 | 담합 인지와 대응 |
| 7 | 스나이퍼의 밤 | 네덜란드식 | 상대 패턴 읽기 (하강 속도 랜덤) |
| 8 | 그랜드 옥션 | 라운드마다 랜덤 | 종합 |

언락: 이전 스테이지 ★ 이상. 별점: ★ 클리어 / +★ 예산 20% 이상 잔여 / +★ 승자의 저주 0회.

## 진행 저장

서버·계정·localStorage 없이 동작합니다. 진행(별점)은 메모리에만 있으므로,
스테이지 선택 화면의 **💾 진행 내보내기**로 JSON 파일을 받아두고 **📂 진행 불러오기**로 복원하세요.

## 기술

- Vite + React 18 + TypeScript (strict) + Tailwind CSS, 외부 상태 라이브러리 없음 (useReducer + Context)
- 순수 프론트엔드 — 백엔드/DB 없음
- 모든 랜덤은 시드 RNG(mulberry32) 단일 출처: **같은 시드 = 같은 판 재현**
- 경매 룰·수치 로직(`src/engine/`)은 React를 모르는 순수 함수이며 전부 Vitest로 테스트
- AI(`src/bidders/`)는 플레이어의 감정치나 입력을 절대 훔쳐보지 않습니다 (공정성)

자세한 설계 명세는 [CLAUDE.md](CLAUDE.md)에 있습니다.
